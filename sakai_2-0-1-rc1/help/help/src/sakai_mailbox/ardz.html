Using Sakai 2.0, how can I set the alias for an Email Archive?

If you have a maintain role, you can create an easy-to-remember alias address for your Email Archive by following these steps:

  1. In the menubar of the relevant course site or project site, click Email Archive.

  2. Click Options.

  3. In the "Site Email Address" field, type the alias you want to use as the initial part of the email address for the archive (i.e., the part before the  @  symbol).

  4. Click Update Options.

  5. You will now see your aliased address as well as the autogenerated address on the main Email Archive screen, for example: Email sent to the following addresses will be archived and sent to participants: german112@oncourse.iu.edu 1105113863261-20909@oncourse.iu.edu

    In this example, german112 was the alias entered in step 3.

If you want to set an alias address for multiple sites you maintain, you can easily do so from Worksite Setup:

  1. In the menubar of My Workspace, click Worksite Setup.

  2. Place a checkmark next to the site for which you want to set an alias, and then click Revise.

    Note: You can only revise settings for one site at a time.

  3. Click Edit Tools.

  4. If it is not already selected, place a checkmark next to Email Archive, and then click Continue.

  5. In the "Site email address:" field, type the alias you want to use.

  6. Click Continue, and then click Finish.

    If you want to set an alias for another site, click Return to Sites List, and then continue from step 2.

Note: You can access the above options from within a site you maintain by clicking Site Info.
