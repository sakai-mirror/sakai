Navigo CVS Best Practices

  1. Daily Routine
    1. Developers should begin their day by retrieving the latest updates from the repository.  This is performed with cvs update:
      1. CVS: cvs update -P -d
      2. Eclipse: Team → Update
    2. Developers should end their day by checking in all code that meets the following criteria:
      1. All code must compile.
      2. Code should not break existing running code (i.e. some basic unit testing should be performed prior to check in).
      3. This is performed with cvs commit:
        1. CVS: cvs commit
        2. Eclipse: Team → Commit
