Using Sakai 2.0, how do I access files in Mac OS X using WebDAV?

WebDAV stands for "Web-based Distributed Authoring and Versioning". It is a set of extensions to the HTTP protocol which allows users to collaboratively edit and manage files on remote web servers.

Mac OS X 10.4 (Tiger)

To use WebDAV to access files in Mac OS X 10.4 (Tiger), follow these steps:

  1. In the Finder, press Cmd-k . (Alternatively, from the Go menu, choose Connect to Server.)

  2. In the "Server Address:" field, enter the appropriate text:

    • If you know the correct URL, you can connect to your personal resource space by entering it into the "Internet or network address" field. Contact your site administrator for the URL's format; it will likely include your username.

    • Similarly, you can connect to the Resources for a specific course, and upload and download files to and from Resources, if you know the correct URL and the unique site ID number. The ID number is the sequence of numbers appearing immediately at the end of your site's URL. For example, once you have clicked the tab for a particular site, the URL field in your web browser will contain something similar to this: https://oncoursecl.iu.edu/portal/site/123456789-123

      The site ID in this case would be 123456789-123. To save a site ID number, while you are in the site, click File, then either Save as... or Save page as... . Make sure the site ID number is in the "File name:" field, and then save it to a place where you can easily retrieve it.

  3. Optionally, you may click the + (plus sign) to the right of the server address after you enter it, and the server will be added to your list of favorite servers.

    When you log in, you can choose to have the password remembered in your Keychain. If you do this, you will not be prompted for a password when you log into your personal Resources or the Resources for any site to which you have access. Do not do this unless your computer is secured from use by others.

You may now drag and drop folders between your hard drive and the Resources to which you just connected.

Mac OS X 10.3 (Panther) and earlier

Using WebDAV to access files in Mac OS X 10.3 (Panther) and earlier requires a third-party client called Goliath. You can download a free copy of Goliath from:

http://www.webdav.org/goliath/installs/Goliath1_0_1.dmg.gz
  1. Once you have downloaded the file, double-click it to decompress it. Then, double-click the resulting .dmg file. A disk image named Goliath 1.0.1 should now be available.

  2. Drag and drop the Goliath folder contained within the disk image to your hard disk.

  3. Open the Goliath application and follow the appropriate instructions below:

    • If you know the correct URL, you can connect to your personal resource space by entering it into the "Internet or network address" field. Contact your site administrator for the URL's format; it will likely include your username.

    • Similarly, you can connect to the Resources for a specific course, and upload and download files to and from Resources, if you know the correct URL and the unique site ID number. The ID number is the sequence of numbers appearing immediately at the end of your site's URL. For example, once you have clicked the tab for a particular site, the URL field in your web browser will contain something similar to this: https://oncoursecl.iu.edu/portal/site/123456789-123

      The site ID in this case would be 123456789-123. To save a site ID number, while you are in the site, click File, then either Save as... or Save page as... . Make sure the site ID number is in the "File name:" field, and then save it to a place where you can easily retrieve it.


  4. The first time you attempt to connect, you may be prompted to accept a certificate for the site. Click OK if prompted.
