Navigo Runtime Configuration

Navigo's runtime environment is configured by a number of externalized configuration files. 

  1. General Settings
    1. The location of the general settings is defined in <WEBAPP>/WEB-INF/web.xml.
    2. You will find two context parameters. The parameters should point to the base directory for each respective parameter.
      1. PathToSecurity
        1. This path houses authentication settings (e.g. database usernames and passwords).
        2. In production environments, developers do not have access to this path.
      2. PathToSettings
        1. This path houses a variety of settings. Some of the common settings you will find here:
          1. Log4j.properties
          2. Database connection settings (i.e. not usernames or passwords).
    3. The paths contain an environment abstraction at the end of the string. These environment names will be replaced by an ant task included in the dist task. Supported environment names:
      1. dev - Development
      2. unt - Unit Test
      3. reg - Regression Test
      4. stg - Staging
      5. trn - Training
      6. prd - Production
    4. Files will be located via standard Java package notation below the base directory:
      1. <PathToSecurity>/org/navigoproject/security
      2. <PathToSettings>/org/navigoproject/settings
    5. Properties stored in these paths can be easily accessed using the class org.navigoproject.settings.PathInfo.
  2. Logging
    1. All application logging is accomplished with Log4j.  As documented in "General Settings", log4j is configured using <PathToSettings>/org/navigoproject/settings/log4j.properties.
    2. Output log file locations are defined in this file. Normally, they are logged to the output folder /opt/logs/<ENV>/<APP_CODE>.
      1. <ENV> is the application environment (dev, reg, prd, etc.).
      2. <APP_CODE> is an institution specific code assigned to the application (a.k.a. code name).
      3. Feel free to change the output folder path to any directory you deem suitable.
  3. OKI OSID Bindings
    1. Implementations of the OKI OSID interfaces are declared in the file ManagerImplBindings.properties.
    2. Currently, ManagerImplBindings.properties is included in the source code in the package org.navigoproject.settings.
    3. This configuration is used by the class org.navigoproject.osid.OsidManagerFactory to provide an easy way for objects to obtain an instance of an OSID interface.
    4. It is likely the location of this file will be moved to the PathToSettings mechanism in the near future.
  4. Application Bootstrapping
    1. The application is bootstrapped and initial runtime values are set by the class org.navigoproject.settings.InitServlet.
    2. This class currently performs four functions:
      1. Initialize PathInfo class.
      2. Initialize Log4j logging system.
      3. Initialize OJB framework.
      4. Initialize OKI OSID implementation.
  5. OJB
    1. OJB is currently being used to persist some data in Navigo. It is expected that this persistence code be replaced with EJB persistence.
    2. OJB is configured through three files:
      1. OJB.properties
      2. repository-nav.xml
      3. repository.dtd
    3. These files are be located in the default package (root package) in the source tree (i.e. src).
  6. STXX
    1. The stxx.properties file is located in the default package (root package) in the source tree (i.e. src).
